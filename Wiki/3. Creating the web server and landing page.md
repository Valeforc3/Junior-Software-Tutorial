## This project will be a next.js application, built on React.

To create a react app, you need to create the following folders and files in your repository:

- 📁 app
    - layout.jsx
    - page.jsx
    - 📁 components
        - Footer.jsx
        - Header.jsx
    - 📁 Data
        - Data1.jsx
        - Data1.json
        - Data2.jsx
        - Data2.jsx
- 🔧 .env.development
- 🔧 .env.production
- 🔧 .gitignore
- 🔧 package.json

The data folder is not mandatory for Node.JS, but it is a very helpful tool to implement, as it allows for centralised data management.

### Breaking down the files:
1. app
    - this has the files that make up the app / website itself as you experience it
2. layout.jsx
    - this file is your "stylesheet", it tells all other files how to present themselves with colours, sizes, etc.
3. page.jsx
    - Every "page" within a react app is a folder with a "page.jsx" file. this one is for the landing page.
4. components
    - these are reusable "elements", that can be referenced in multiple pages
5. header.jsx
    - this is your code for your page header (the header is the segment at the top of your page, usually has a logo and menus)
6. foooter.jsx
    - the opposite of the header, it sits at the bottom of each page
7. data
    - this folder holds all of your application's data, that could be used in several locations within your app
8. package.json
    - this is the file that tells our server hosting platform (in this case, Vercel), how to build the server itself.
